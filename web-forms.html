WuXi AppTec - Web Forms
Built with C#, JavaScript, jQuery, HTML, CSS, SQL, ASP.NET
There are multiple web forms that I created which clients use to order tests and fill out information regarding the tests ordered. Only specific tests can be ordered on each form. Once the client is done filling out the form, he or she prints it, scans it, and sends it to the appropriate location. There is also an option to save the form as a PDF instead of printing it out. Click here to see all of the different web forms: http://us.wuxiapptec.com/biologics/ordering-testsclient-portal/
Ordering Tests
The first page of the web forms have to do with entering in the test to be ordered. If it is a test that can be ordered on the form, then it will display the Test Name after typing in the Test Code. Sometimes a test that can be ordered on the form will not show the Test Name because the Quote Number and Account Number for that test needs to be entered. If the test code does not exist or cannot be ordered on the form, then it will display the red text shown in the third row in the Test Name column. There is a check box at the bottom of the page that will display a new page to enter in more test codes if it is checked. The content in the form is remembered so if the browser you had the form opened in is closed and you come back to it, the same content will display from before. The Reset Form button is used to set all the fields back to empty, which clients use this when they come back to the form at a later time and want to enter in new tests and information.
Information for Test
This is an example of a page that has information to fill out related to the tests ordered. Some dropdowns and check boxes display more content depending on what is selected or clicked. For example in the Extraction Conditions dropdown, if the selection Other is chosen, then the text Specify and a text box are shown below the dropdown. There are some cases on the forms where if a specific test is ordered, it will display a new section to be filled out. The sections that appear can be based on the type of category, department, etc. of a test code.
Printing the Form
Here is an example of the page that contains the Print/Save button. It is recommended for the client to click this button instead of using the browser to print the form because once the button is clicked, it checks to see if any fields have not been filled out that are required. If the client prints from the browser instead of using the button, the message shown below is shown to remind the client to use the button instead when printing or saving the form as a PDF.
Required Fields
When the Print/Save button is clicked and there are fields that are required and do not have content, then the message displayed below in the first image is shown on the form. If the OK button is clicked, the print window will open and there will be a WARNING message shown at the top of the page that will be printed or saved as a PDF, which is shown in the left image. Whereas, if the OK button is clicked, then the client is brought to the top of the first page and all required fields that are missing content are highlighted in yellow. Some fields, like the ones shown in the left image, are always required regardless of what tests are entered into the form. Other fields are only required if specific tests are entered. If a change is made, such as typing information into a field, and the browser was used to print the form instead of the Print/Save button, then the form will print with the second image shown below at the top of it.
